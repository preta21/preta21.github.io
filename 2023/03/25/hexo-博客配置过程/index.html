创建 Github Page

以下图片来源 链接

  1. 在github上创建新的分支

  2. 分支的命令为 <用户名>.github.io

  3. 分支创建为public模式

  4. 新建分支后，新建指定名称的文件 index.html，用来检查github分支的网页显示是否正常。

  5. 然后，在该分支的页面，点击 setting->Pages，查看github生成的网页链接

    如果没有显示链接，就刷新，等一会儿。
    当然，我记得是在如上页面中，要选择

    Source->Branch->main
    /(root)
    才能成功生成链接。链接一般是 https://[yourname].github.io/。yourname就是自己github首页显示的名称。
    点开链接，查看网页显示是否正常，和index.html中一样。

  6. 回到分支的<>Code界面，选择该分支的 ssh 方式链接，复制链接。选择ssh的链接能在git clone时避免出现https链接的TSL握手问题。

拿到 github access token

此token的位置在
github首页->头像->Settings->Developer settings->Personal access tokens

->Tokens页面，点击 Generate new token->(classic)

注意，在创建 token 的界面，在Select scopes中一定要勾选 repo属性，这样才能授权本地的博客应用将内容部署到远程分支。

其中Note是必填项，也就是该token的名称。

将本地博客部署到 github page

安装部署到git的工具

修改 hexo-blog/_config.yml

注意，这里一定要配置 branch，且必须指定为main。
因为新的github上的默认分支为 main，但是本地默认的分支名称为master。
如果忘了改配置，在远端就会创建一个master分支。这样的话无法合并，那就先在github删除远端master，在本地将branch配置为main再部署一次。
token就是前面拿到的github access token。

还要注意一点，当前github已经放弃通过https输入用户名密码的方式来访问，只能通过ssh来访问。
这也是为什么我在前面使用github上的ssh链接。
当然，本地若想通过ssh访问github，除了拿到ssh链接，还要将本地的密钥放到github上。
